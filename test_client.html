Django-Managed OAuth Flow

OAuth Flow:

  1. Start by going to your Django app running on port 8000
  2. When you need to authenticate, your Django app will redirect to the MCP server's authorization endpoint
  3. The MCP server will then redirect back to your Django app with an authorization code
  4. Your Django app will exchange this code for an access token
  5. Once you have the access token, you can use it in this test client to make API calls

Important Notes:

  • This test client should NOT be used to initiate the OAuth flow directly
  • The OAuth flow should be handled entirely by your Django app
  • This client is only for testing the MCP API endpoints once you have a valid access token
